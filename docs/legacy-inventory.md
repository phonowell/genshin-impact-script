# Legacy Inventory

## Purpose

This file freezes the current Coffee/AHK v1 system as a behavior reference for the AHK v2 rewrite.

## User Features

Core features documented in the repo:

- automatic fishing
- automatic pickup
- combat assist
- skill timer
- one-button tactics and party switching
- recorder and replayer
- dialogue skip
- window positioning
- background mute

## Hotkeys

Source: `doc/keymap.md`

- hold `LButton` or side button: custom action
- `RButton`: exit current menu
- `Alt+F4`: exit script
- `Ctrl+F5`: reload script
- `F11`: toggle fishing
- `F12`: scan party
- `~`: switch character and use elemental skill
- `1..5`: switch character
- `Alt+1..5`: switch character and use elemental burst
- `Alt+W`: toggle auto-forward
- `W` or `S`: stop auto-forward
- hold `F`: fast pickup or skip dialogue
- `Alt+F`: toggle auto-pickup
- `Alt+Z`: toggle auto-gadget
- `Alt+Space`: toggle better jump
- arrow keys: move camera
- `Win+Arrow`: move game window
- `Ctrl+NumpadDot`: start or stop recording
- `Ctrl+Numpad0..9`: replay a slot

## Runtime Shape

Legacy runtime characteristics:

- source language: CoffeeScript
- compiler: `coffee-ahk`
- helper runtime: `shell-ahk`
- generated target: AutoHotkey v1
- assembly model: side-effect imports into a shared global namespace
- boot model: sorted import list plus `misc.coffee` bootstrap

## Environmental Assumptions

- game runs in 16:9
- resolution is at least 1600x900
- script runs as admin
- no image filters
- some functions require English input mode
- party scan is required before some skill-timer flows

## Rewrite Priorities

Priority 1:

- config loading
- window detection
- screenshot and color reads

Priority 2:

- scene and state detection
- party and character resolution

Priority 3:

- movement, pickup, camera
- combat, skill, tactic

Priority 4:

- recorder, replayer, fishing, mute, HUD

## Known Risks

- old behavior depends on hidden global state
- some values come from generated or external data
- graphics and pixel thresholds are fragile
- input timing is sensitive to runtime differences
- there is no strong regression harness yet

## Rewrite Policy

- keep this inventory as the behavior checklist
- do not mirror the old file structure blindly
- port features only after the required lower layer is stable
